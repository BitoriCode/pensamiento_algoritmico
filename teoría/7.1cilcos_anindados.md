# Ciclos anidados.

¿Alguna vez han intentado buscar un nombre en una lista de asistencia organizada por salones? Primero van salón por salón, y dentro de cada salón revisan nombre por nombre. Eso, en esencia, es un ciclo anidado: **un bucle dentro de otro**.

Estas notas recogen lo fundamental del tema para el curso. No pretenden ser un manual exhaustivo — son más bien el tipo de apuntes que me hubiera gustado tener cuando vi el tema por primera vez.

## Antes de arrancar

Para sacarle provecho a esto necesitan manejar:

- Variables y tipos básicos
- Condicionales (`if`, `elif`, `else`)
- Al menos lo básico de `for` y `while` por separado

Si todavía no se sienten cómodos con un `for` sencillo, repasen eso primero. Acá vamos a combinarlos y la cosa se pone interesante.

## ¿Qué es un ciclo anidado?

La idea es sencilla: un ciclo que vive dentro del cuerpo de otro ciclo. Lo que pasa es que **por cada vuelta del ciclo de afuera, el ciclo de adentro se ejecuta completico**. Y eso, cuando no se entiende bien, genera mucha confusión.

Piénsenlo así: si el ciclo exterior da 5 vueltas y el interior da 3, en total el cuerpo interior se ejecuta 5 × 3 = 15 veces. No 5, no 3, no 8. **Quince**.

Para problemas con matrices, la lectura es casi siempre la misma:
- Ciclo exterior → filas
- Ciclo interior → columnas

O en palabras: *"recorro una fila completa antes de pasar a la siguiente"*.

### Repaso rápido: `for` vs `while`

Antes de meter ciclos dentro de ciclos, asegurémonos de que la base está clara:

| | `for` | `while` |
|---|---|---|
| ¿Cuándo usarlo? | Cuando sabés cuántas veces iterar o tenés algo que recorrer (lista, rango) | Cuando no sabés cuántas veces, sino que dependés de una condición |
| Ejemplo mental | "Para cada estudiante en la lista, hacer..." | "Mientras el usuario no ingrese un número válido, pedir de nuevo..." |
| Peligro clásico | Pasarse de índices si se usan mal | Ciclo infinito si se olvida actualizar la condición |

En la práctica, para ciclos anidados casi siempre vamos a usar `for` con `range()`, porque generalmente sabemos las dimensiones de lo que estamos recorriendo.

### Cosas que confunden al principio.

**La variable del `for` se actualiza sola.** Si escriben `for i in range(5)` e intentan cambiar `i` dentro del cuerpo pensando que eso altera el recorrido... no funciona. El `for` siempre toma el siguiente valor del `range`, sin importar lo que le hagan a `i` adentro.

**La variable NO desaparece después del ciclo.** En Python, después de un `for`, la variable queda viva con el último valor que tuvo. Esto sorprende bastante, y a veces causa bugs difíciles de encontrar cuando reutilizan nombres sin darse cuenta.

**`break` solo rompe UN nivel.** Esto es clave. Si tienen un `for` dentro de otro `for` y hacen `break`, solo salen del `for` interior. El exterior sigue como si nada. Para salir de ambos hay que usar una bandera (una variable booleana) o meter el código en una función y usar `return`.

**`continue` salta a la siguiente vuelta del ciclo actual**, no del exterior. Y el `else` de un bucle (rara vez lo van a necesitar ahora, pero existe) se ejecuta si el bucle termina sin que haya pasado un `break`.

### ¿Qué tan "caro" es anidar ciclos?

No les voy a meter teoría formal de complejidad algorítmica porque eso es tema de semestres más avanzados. Pero la intuición es importante:

- Un ciclo solo → si recorre `n` cosas, hace `n` operaciones.
- Dos ciclos anidados → si ambos recorren `n` cosas, son `n × n` operaciones. Con `n = 100`, ya van 10,000. Con `n = 1000`, un millón.
- Tres niveles → `n × n × n`. Las cosas escalan rápido.

La regla : **cada nivel de anidamiento multiplica el trabajo**. Por eso, si se encuentran escribiendo cuatro o cinco niveles de ciclos, probablemente necesiten repensar el enfoque (o al menos preguntarse si hay una forma más simple).

## Ejemplos paso a paso

### El más básico: coordenadas

Empecemos con algo simple para ver cómo se mueve la ejecución:

```python
filas = 2
columnas = 3

for i in range(filas):
    for j in range(columnas):
        print(f"i={i}, j={j}")
```

Léanlo en voz alta: *"para cada fila, recorro todas las columnas"*. Si hacen la traza:

| Paso | i | j |
|---:|---:|---:|
| 1 | 0 | 0 |
| 2 | 0 | 1 |
| 3 | 0 | 2 |
| 4 | 1 | 0 |
| 5 | 1 | 1 |
| 6 | 1 | 2 |

Fíjense que `j` cambia rápido (se "reinicia" en cada fila) e `i` cambia lento. Eso **siempre** es así: el ciclo interior gira más rápido.

### Tabla de multiplicar

El clásico. Lo incluyo porque es muy fácil de visualizar y verificar a mano:

```python
n = 5

for i in range(1, n + 1):
    for j in range(1, n + 1):
        print(f"{i*j:4}", end="")
    print()
```

Dos cosas para notar:
- El `end=""` evita el salto de línea dentro del ciclo interior (así los números quedan en fila).
- El `print()` solito al final es el que hace el salto de línea al terminar cada fila. Muchos se confunden con esto al principio — pruébenlo quitando ese `print()` para ver qué pasa.

### Triángulo de asteriscos (el ciclo interior depende del exterior)

Este tipo de ejercicios parecen de relleno, pero son buenísimos para entender una idea clave:

```python
altura = 4

for fila in range(1, altura + 1):
    for k in range(fila):        # ← acá está la gracia
        print("*", end="")
    print()
```

Salida:
```
*
**
***
****
```

¿Qué tiene de especial? El `range(fila)`, no `range(altura)`. **El ciclo interior cambia de tamaño en cada vuelta del exterior.** En la fila 1 imprime 1 asterisco, en la fila 2 imprime 2, y así. Si les pregunto "¿cuántos asteriscos se imprimen en total?", la respuesta es $1 + 2 + 3 + 4 = 10$, o en general $\frac{n(n+1)}{2}$.

### Recorrer matrices (donde esto se vuelve realmente útil)

En Python, una "matriz" es simplemente una lista de listas:

```python
matriz = [
    [1, 2, 3],
    [4, 5, 6],
]
```

Y para trabajar con ella necesitamos dos ciclos: uno para las filas y otro para los valores de cada fila. Veamos los casos más comunes:

**Sumar cada fila:**

```python
for i, fila in enumerate(matriz):
    suma = 0
    for valor in fila:
        suma += valor
    print(f"Fila {i}: suma = {suma}")
```

`enumerate` les da el índice y el valor al mismo tiempo. Es mucho más limpio que andar con `range(len(...))`.

> **Ojo:** el `suma = 0` tiene que estar **dentro** del primer `for`, no afuera. Si lo ponen afuera, están sumando toda la matriz junta en vez de fila por fila. Este error lo veo en casi todos los parciales.

**Sumar cada columna (este confunde más):**

```python
filas_n = len(matriz)
cols_n = len(matriz[0])

for j in range(cols_n):
    s = 0
    for i in range(filas_n):
        s += matriz[i][j]
    print(f"Columna {j}: suma = {s}")
```

Acá el ciclo exterior recorre columnas y el interior recorre filas. Es "al revés" de lo habitual, y por eso cuesta. El truco: *"fijo la columna y bajo por todas las filas"*.

**Encontrar el máximo y en qué posición está:**

```python
matriz = [
    [3, 1, 7],
    [2, 9, 4],
    [8, 0, 5],
]

max_val = matriz[0][0]
pos = (0, 0)

for i, fila in enumerate(matriz):
    for j, valor in enumerate(fila):
        if valor > max_val:
            max_val = valor
            pos = (i, j)

print(f"Máximo: {max_val} en posición {pos}")
```

Arranco asumiendo que el máximo es el primer elemento, y después recorro todo para ver si alguien le gana. Simple.

**Buscar un valor y parar cuando lo encuentre:**

```python
objetivo = 50
encontrado = False
pos = None

for i, fila in enumerate(matriz):
    for j, valor in enumerate(fila):
        if valor == objetivo:
            encontrado = True
            pos = (i, j)
            break
    if encontrado:
        break

if encontrado:
    print(f"Está en {pos}")
else:
    print("No está")
```

Acá se ve lo del `break` que mencioné antes: rompe el ciclo interior, pero para salir del exterior necesitamos checar la bandera `encontrado`. Es un poco incómodo, pero es el patrón estándar. Si estuvieran dentro de una función, podrían meter un `return` directo y ahorrarse la bandera.

### Triple anidamiento: coordenadas 3D (para ver cómo escala)

Solo para que dimensionen el crecimiento:

```python
n = 2

for x in range(n):
    for y in range(n):
        for z in range(n):
            print(f"({x},{y},{z})")
```

Con `n=2` son 8 combinaciones ($2^3$). Con `n=10` serían 1,000. Con `n=100`, un millón. Cada nivel que agregan **multiplica** la cantidad de trabajo.

### Para los más curiosos: sopa de letras (cuatro niveles)

Este ejemplo es más avanzado. Si les parece mucho, pueden saltarlo y volver después — no pierden nada esencial.

La idea: en un tablero de letras, buscar si una palabra aparece en alguna dirección (derecha, izquierda, arriba, abajo).

```python
cuadricula = [
    ['c', 'a', 's', 'a'],
    ['x', 'x', 'x', 'x'],
    ['x', 'x', 'x', 'x'],
    ['a', 's', 'a', 'c'],
]
palabra = "casa"

filas = len(cuadricula)
cols = len(cuadricula[0])
direcciones = [(0, 1), (0, -1), (1, 0), (-1, 0)]

hallado = None

for i in range(filas):                     # fila de inicio
    for j in range(cols):                  # columna de inicio
        for dr, dc in direcciones:         # dirección
            ok = True
            for k in range(len(palabra)):  # letra por letra
                ii, jj = i + dr*k, j + dc*k
                if not (0 <= ii < filas and 0 <= jj < cols):
                    ok = False
                    break
                if cuadricula[ii][jj] != palabra[k]:
                    ok = False
                    break
            if ok:
                hallado = (i, j, dr, dc)
                break
        if hallado:
            break
    if hallado:
        break

print("Resultado:", hallado)
```

Son cuatro niveles (fila × columna × dirección × letra). Es un buen ejemplo de que a veces sí se necesitan varios niveles, pero son casos específicos. Para la mayoría de problemas de este curso, con dos niveles alcanzan de sobra.

## Ejercicios para practicar

Van de menor a mayor dificultad. **Intenten resolverlos tapando la solución.** Hacer la traza a mano antes de correr el código es lo que realmente les va a enseñar — copiar y pegar no sirve de nada.

### 1. Rectángulo de caracteres

Dado un `alto` y un `ancho`, impriman un rectángulo de `#`.

```python
alto = 3
ancho = 6

for fila in range(alto):
    for col in range(ancho):
        print("#", end="")
    print()
```

### 2. Triángulo numérico

Impriman un triángulo donde la fila `i` muestre el número `i` repetido `i` veces:

```python
n = 5

for i in range(1, n + 1):
    for _ in range(i):
        print(i, end=" ")
    print()
```

¿Cuántos números se imprimen en total? Son $1 + 2 + ... + n = \frac{n(n+1)}{2}$. Para $n = 5$, sale 15.

### 3. Contar pares con suma par

Cuenten cuántos pares `(i, j)` con `0 ≤ i, j < n` tienen suma par:

```python
n = 5
contador = 0

for i in range(n):
    for j in range(n):
        if (i + j) % 2 == 0:
            contador += 1

print("Cantidad:", contador)
```

### 4. Construir la matriz identidad

Generen una matriz identidad de tamaño `n` (unos en la diagonal, ceros en el resto):

```python
n = 4
matriz = []

for i in range(n):
    fila = []
    for j in range(n):
        fila.append(1 if i == j else 0)
    matriz.append(fila)

for fila in matriz:
    print(fila)
```

### 5. Sumas por filas y por columnas

Dada una matriz, impriman la suma de cada fila y la suma de cada columna por separado:

```python
matriz = [
    [1, 2, 3],
    [4, 5, 6],
]
filas_n = len(matriz)
cols_n = len(matriz[0])

print("--- Filas ---")
for i, fila in enumerate(matriz):
    print(f"Fila {i}: {sum(fila)}")

print("--- Columnas ---")
for j in range(cols_n):
    s = 0
    for i in range(filas_n):
        s += matriz[i][j]
    print(f"Columna {j}: {s}")
```

### 6. Máximo y su posición

Encuentren el valor más grande en una matriz y digan en qué posición está:

```python
matriz = [
    [3, 1, 7],
    [2, 9, 4],
    [8, 0, 5],
]

max_valor = matriz[0][0]
pos = (0, 0)

for i, fila in enumerate(matriz):
    for j, valor in enumerate(fila):
        if valor > max_valor:
            max_valor = valor
            pos = (i, j)

print(f"Máximo: {max_valor} en {pos}")
```

### 7. Búsqueda con parada temprana

Busquen un número en una matriz. Si lo encuentran, impriman dónde está y paren. Si no, digan que no está:

```python
matriz = [
    [10, 20, 30],
    [40, 50, 60],
]
objetivo = 50

encontrado = False
pos = None

for i, fila in enumerate(matriz):
    for j, valor in enumerate(fila):
        if valor == objetivo:
            encontrado = True
            pos = (i, j)
            break
    if encontrado:
        break

if encontrado:
    print(f"Está en {pos}")
else:
    print("No está")
```

## Errores que aparecen una y otra vez

Después de varios semestres viendo los mismos problemas, estos son los clásicos:

**Mezclar tabs y espacios en la indentación.** Python no perdona esto. Van a ver `TabError` o `IndentationError` y el código parece estar bien a simple vista. Consejo: configuren VS Code para usar siempre 4 espacios (busquen "Tab Size" en la configuración y activen "Insert Spaces"). PEP 8 recomienda exactamente eso.

**Usar la misma variable en el ciclo exterior e interior.** Si escriben `for i in range(3): for i in range(3):` están pisando la `i` de afuera con la de adentro. Usen nombres diferentes. `i` y `j` sirven, pero `fila` y `col` son todavía mejor porque dicen qué representan.

**No reiniciar el acumulador.** Este es el más frecuente y el que más puntos cuesta:

```python
# MAL: suma todo junto
suma = 0
for fila in matriz:
    for valor in fila:
        suma += valor
    print(suma)  # imprime valores acumulados, no la suma de cada fila

#  BIEN: reinicia en cada fila
for fila in matriz:
    suma = 0     # ← esto tiene que ir ACÁ
    for valor in fila:
        suma += valor
    print(suma)
```

**Creer que `break` saca de todos los ciclos.** Ya lo dije dos veces pero lo repito porque sigue pasando: `break` solo rompe el ciclo más inmediato. Si necesitan salir de dos niveles, usen una bandera o una función con `return`.

**Escribir `range(len(lista))` cuando `enumerate` es más limpio.** No es un error que rompa el programa, pero el código queda innecesariamente enredado. Comparen:

```python
# Funciona pero es feo
for i in range(len(fila)):
    print(fila[i])

# Más claro
for valor in fila:
    print(valor)

# Si necesitan el índice también
for i, valor in enumerate(fila):
    print(i, valor)
```

## Para cerrar

Algunas recomendaciones que ojalá alguien me hubiera dado antes:

- **Hagan trazas a mano.** Papel y lápiz, tabla con las variables, paso por paso. No hay nada que reemplace eso para entender qué está pasando.
- **Prueben con ejemplos chiquitos.** No empiecen con matrices de 100×100. Una de 2×3 donde puedan verificar a mano es infinitamente más útil para depurar.
- **Lean el código en voz alta.** "Para cada fila, para cada columna, hago tal cosa" — si la frase suena lógica, probablemente el código esté bien.
- **Si tienen más de 3 niveles de anidamiento**, pregúntense si no pueden sacar parte de la lógica a una función aparte. No por eficiencia, sino porque nadie quiere leer cinco niveles de indentación.
- Y sobre todo: **no se frustren si al principio cuesta.** Los ciclos anidados son el primer salto real de dificultad en programación. Si los entienden bien, lo que viene después (funciones, recursión) les va a ser mucho más llevadero.
